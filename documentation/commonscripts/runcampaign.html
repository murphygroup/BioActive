About BioActive's runCampaign2

runCampaign2.py integrates the functionality of a potentially highly variable ESS (experimental space specification) with swappable modular functionality into a single, common active learning framework delivering a reliable model, minimizing experimentation.

File Structure

Initialization

runCampaign2.py initializes the campaignObject before any experimentation has begun. The following call initializes the model for the campaign by attach to it the attributes later required by the modelers. However, each campaign's model is different and some might require some additional attributes. These specific attributes should be initialized in the modeler file. See the modeler page for more information.

initModel.main(campaignObject)

and the following call initializes the associated variables for the pool-based campaigns. Does nothing for other campaigns.

campaignObject.initAvars()

Modeling Phase

The "modeling phase" is a while loop that first models the data chosen (none for the first round of iteration), then stores information obtained from modeling the data. The loop ends once the specified goal has been attained, which is checked every iteration through the goalTether function.
